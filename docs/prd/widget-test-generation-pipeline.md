# Widget Component Test Generation Pipeline — PRD

## Problem

Widget specs (`.widget` files) declare rich testable structure — FSM transitions, connect bindings, ARIA attributes, keyboard bindings, focus management, props, invariants, and composition. We have:

1. **WidgetComponentTest concept + handler** — builds language-neutral test plans from widget specs
2. **5 framework-specific renderers** — React, Playwright, Vue, Svelte, Vanilla DOM
3. **3-way merge infrastructure** — exists in `generate-all-tests.ts` for concept conformance tests

But these pieces aren't wired together into an automated pipeline. Today, generating widget component tests requires manual invocation. There's no script that scans `.widget` files, generates tests, and preserves developer patches on regeneration.

## Goals

1. **Automated widget test generation** — scan all `.widget` files, generate framework-specific test files, with the same developer experience as `npx tsx scripts/generate-all-tests.ts`
2. **3-way merge with developer patches** — complex widget behaviors (drag-and-drop, async loading, portal positioning, animations) require hand-written tests that must survive regeneration
3. **Parity with concept conformance tests** — same `@clef-patched` markers, same baseline storage, same merge behavior

## Design

### Generation Pipeline

```
.widget file
  → widget-spec-parser.ts (parse to WidgetManifest)
  → WidgetComponentTest/buildPlan (extract 8 test categories)
  → framework renderer (React/Playwright/Vue/Svelte/Vanilla)
  → generated test file (with merge markers)
```

### Script: `scripts/generate-widget-tests.ts`

Extends or mirrors `generate-all-tests.ts` with:

```typescript
// 1. Discover all .widget files
const widgetFiles = glob('surface/**/*.widget', 'repertoire/**/*.widget');

// 2. For each widget, parse and build test plan
for (const file of widgetFiles) {
  const manifest = parseWidgetFile(readFileSync(file));
  const plan = buildWidgetTestPlan(manifest);

  // 3. Generate test for each configured target framework
  for (const target of config.widgetTestTargets) {
    const renderer = renderers[target]; // react, playwright, vue, etc.
    const testCode = renderer(plan, manifest.name, importPath);

    // 4. 3-way merge with existing file
    const testPath = `generated/widget-tests/${manifest.name}.${target}.test.ts`;
    writeWithMerge(testPath, testCode); // reuses existing merge infra
  }
}
```

### 3-Way Merge (reuse existing infrastructure)

The merge system from `generate-all-tests.ts` (lines 118-540) already handles:

1. **First generation** — write file + store baseline
2. **No changes** — skip (unchanged output)
3. **Generator changed, user didn't patch** — overwrite safely
4. **User patched, generator unchanged** — preserve patches
5. **Both changed** — 3-way merge, warn on conflicts

Widget tests use the same system. Developer patches are marked:

```typescript
// Auto-generated by WidgetComponentTest from check-list.widget
// DO NOT EDIT above the @clef-custom-tests marker

describe('CheckList', () => {
  // ... generated tests ...

  // ─── @clef-custom-tests ─────────────────────────
  // Add custom tests below. These survive regeneration via 3-way merge.
  // Mark the file with @clef-patched if you modify generated tests above.

  describe('Custom: drag reorder', () => {
    it('dragging item 2 above item 1 reorders the list', async () => {
      // Hand-written test for complex behavior
    });
  });
});
```

### Configuration

In `clef.config.ts` or `widget-test.config.ts`:

```typescript
export default {
  // Which .widget directories to scan
  widgetPaths: ['surface/**/*.widget', 'repertoire/**/*.widget'],

  // Which frameworks to generate tests for
  targets: ['react', 'playwright'], // default: react + playwright

  // Output directory
  outputDir: 'generated/widget-tests',

  // Baseline storage for 3-way merge
  baselineDir: '.clef-baselines/widget-tests',
};
```

### CLI Integration

```bash
# Generate widget component tests
npx tsx scripts/generate-widget-tests.ts

# Generate for specific widget
npx tsx scripts/generate-widget-tests.ts --widget check-list

# Generate for specific framework
npx tsx scripts/generate-widget-tests.ts --target playwright

# Force overwrite (ignore patches)
npx tsx scripts/generate-widget-tests.ts --force
```

## Test Categories (from WidgetComponentTestPlan)

| Category | Generated From | Test Pattern |
|----------|---------------|--------------|
| `fsm_transitions` | `states { ... }` | Render → interact → assert data-state |
| `connect_bindings` | `connect { ... }` | Render with state → assert attributes |
| `keyboard_bindings` | `accessibility.keyboard` | fireEvent.keyDown → assert transition |
| `focus_management` | `accessibility.focus` | Render → assert activeElement |
| `aria_assertions` | `accessibility.aria` | Render → assert ARIA attributes |
| `props` | `props { ... }` | Render with defaults → assert output |
| `invariants` | `invariant { ... }` | Structured → behavioral test sequences |
| `compose` | `compose { ... }` | Render → assert child widget presence |

## What Developers Patch (complex behaviors generators can't express)

| Behavior | Why Generator Can't Express It |
|----------|-------------------------------|
| Drag and drop | Requires pointer event sequences with coordinates |
| Async loading | Requires `waitFor()` / timing control |
| Animations | Requires `requestAnimationFrame` / CSS transition events |
| Portal positioning | Requires viewport-relative coordinate assertions |
| Virtual scrolling | Requires scroll position + DOM recycling assertions |
| Complex keyboard nav | Grid 2D navigation needs multi-key sequences with state |
| External data | Requires mock API setup and response simulation |
| Responsive layout | Requires viewport resize between assertions |
| Composed widget interaction | Requires rendering real child widgets with mock props |

## Implementation Plan

### Phase 1: Core Pipeline
1. Create `scripts/generate-widget-tests.ts` reusing merge infrastructure
2. Wire WidgetComponentTest handler into the script
3. Default to React + Playwright targets
4. Add `@clef-custom-tests` marker and `@clef-patched` support
5. Store baselines in `.clef-baselines/widget-tests/`

### Phase 2: CLI Integration
1. Add `--widget` and `--target` flags
2. Add to `npx tsx cli/src/main.ts generate` command
3. Add to CI pipeline

### Phase 3: Additional Renderers
1. Wire Vue, Svelte, Vanilla renderers into the pipeline
2. Add per-widget target overrides in config

## Non-Goals

- **Visual regression testing** — screenshots/pixel comparison is out of scope
- **Performance testing** — render timing/memory usage is out of scope
- **E2E testing** — full application flows are out of scope (Playwright tests here are component-level, not page-level)
