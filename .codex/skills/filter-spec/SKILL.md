# FilterSpec

Represent reusable , composable predicate trees that determine which records pass a filter . Each filter holds a serialized FilterNode AST , the set of fields it references ( for pushdown analysis ) , the bound context variables it expects , and its source classification . Filters can be composed algebraically into AND trees , evaluated directly against JSON row arrays , normalized to canonical forms , validated against a schema , and have parameter nodes substituted with concrete values at runtime

## Commands

### create
Register a new named filter with its serialized FilterNode tree and 
 source classification . The filter is immediately available for 
 composition , evaluation , and validation .

**Arguments:** `$0` **name** (string), `$1` **tree** (string), `$2` **sourceType** (string)

### get
Return the filter s tree , source classification , and the field 
 references extracted from the tree at registration time .

**Arguments:** `$0` **name** (string)

### compose
Create a new unnamed filter whose tree is 
 { type : and , conditions : [ a . tree , b . tree ] } . The resulting 
 FilterNode preserves both sub trees . Composing with the identity 
 filter ( type : true ) produces a tree semantically equal to the 
 other operand . Composition is commutative and idempotent .

**Arguments:** `$0` **a** (string), `$1` **b** (string)

### evaluate
Apply the filter s predicate tree to the provided JSON array of 
 row objects and return only the rows that match the predicate . 
 The result is a JSON array string containing the matching rows .

**Arguments:** `$0` **name** (string), `$1` **rows** (string)

### normalize
Return the filter tree converted to the requested normal form 
 ( cnf or dnf ) . The normalized tree is semantically equivalent 
 to the original and can be substituted anywhere the original 
 tree would be used .

**Arguments:** `$0` **name** (string), `$1` **form** (string)

### validate
All field references in the filter tree exist in the provided 
 schema . The filter is safe to apply against data matching that 
 schema .

**Arguments:** `$0` **name** (string), `$1` **schema** (string)

### bind
Replace every { type : param } node in the filter tree with the 
 concrete value from the bindings map that matches the node s 
 parameter name . The bound tree is returned as a serialized 
 FilterNode string ready for evaluation .

**Arguments:** `$0` **name** (string), `$1` **bindings** (string)

### list
Return all registered filters as a JSON array , each entry 
 containing the filter s name , tree , sourceType , and fieldRefs .
